Technical Reference

CLI Argument Reference

The following table lists all arguments accepted by llm_ensemble.py.

Flag / Argument Required Default Description
-m, --models Yes - Comma-separated list (e.g., gemini,codex:gpt-4).
-n, --iterations Yes - Number of runs per model (Integer > 0).
-p, --prompt No* - Input text. Mutually exclusive with -f.
-f, --prompt-file No* - Path to input file. Auto-detects encoding.
-c, --context No - Attach files as context. Supports .txt, .pdf, .docx. Repeatable.
[Positional] No* - Input text if -p is omitted.
-o, --outdir No Outputs/llm_ensemble_... Directory to save all results.
--gemini-model No gemini-3-pro-preview Default model for Gemini provider.
--codex-model No gpt-5.2-codex Default model for Codex provider.
--codex-reasoning No high minimal, low, medium, high, xhigh.
--merge-prompt No (Built-in) Custom instruction for the final merger.
--timeout No 300 Timeout in seconds for each run.
--format No txt Output format: txt or rtf.

* One of prompt, prompt-file, or positional prompt is required.

System Architecture

The Python implementation improves upon the legacy Bash script by introducing true concurrency.

Execution Flow

  1. Validation: The Config object is built from arguments. File paths and dependencies are checked.
  2. Parallel Execution: A ThreadPoolExecutor spawns tasks.
    • Each task is a unique combination of (Model, Iteration).
    • Tasks invoke GeminiRunner or CodexRunner.
    • Output is written atomically to _run_X.txt files.
  3. Merging:
    • All successful result files are read.
    • A "Merge Prompt" is constructed (Instructions + Original Prompt + Candidates).
    • The Merge Model (Codex) generates the final answer.

Output Artifacts

Every run creates a directory containing:

  • prompt.txt: The canonicalized input prompt (UTF-8).
  • [model]_run_[n].txt: The raw output of a specific model iteration.
  • [model]_run_[n].log: Stderr logs (useful for debugging timeouts/crashes).
  • merge_prompt.txt: The exact prompt sent to the merger model.
  • final.txt: The final synthesized answer.
  • final.rtf: (Optional) The formatted document if --format rtf was used.

Troubleshooting

"Prompt file appears to be binary"

The tool detected null bytes in your input file. Ensure it is a valid text file.

Timeout Errors

If models are timing out (default 300s), increase the limit using --timeout 600.

"codex command not found"

The tool relies on the codex binary being in your system PATH. Verify installation.
